# v5.5.2 Release Summary — Faithful Workflows Visual Impact Upgrade

**Date:** April 1, 2026
**Branch:** `main`
**Release type:** Minor — new UI canvas views, no functional or data changes
**Status:** Complete

---

## Overview

Extends the Faithful Workflows page with two new parallel canvas views — **Radial Hub** and **Priority Matrix** — while keeping the original **Classic List** view fully intact. A floating cycle button in the canvas panel lets the counselor switch between all three views instantly; the choice persists across sessions via `localStorage`.

Zero changes were made to the rules engine, recommendation logic, API routes, database schema, or any shared component. All 51 engine integration tests continue to pass. The bundle size increase is +5.7 KB gzipped with no new dependencies.

---

## Why This Matters

The Faithful Workflows engine already surfaces prioritised, explainable, AI-assisted care recommendations in real time. This release upgrades the **visual impact** of that output so counselors can absorb the clinical picture more quickly:

- **Classic List** (original): linear vertical stack — best for methodical top-to-bottom review.
- **Radial Hub** (new): hub-and-spoke diagram — best for a quick overview of which domains are active and how the client presents as a whole.
- **Priority Matrix** (new): 2D triage grid — best for answering "what is both urgent AND in what domain?" at a glance.

All three views share the same underlying data, the same drawer, and the same action outputs.

## New Views

### Radial Hub (`WorkflowCanvasRadial.jsx`)

The client snapshot (name, urgency badge, trend arrow, pending recommendation count) lives inside a **central hub circle**. Its border colour mirrors the client's urgency level (red = critical, orange = high, etc.).

Each active recommendation category becomes a **satellite** connected to the hub by a coloured SVG spoke:

```
                  ┌─────────────┐
                  │ 🚨 SAFETY   │  ← solid red spoke (Safety always at 12-o'clock)
                  └──────┬──────┘
                         │
   ┌──────────────┐      │        ┌──────────────┐
   │ ⚠️ CLINICAL  │──────┼────────│ 🎯 SESSION   │
   └──────┬───────┘  ┌───┴────┐   └──────┬───────┘
          │          │ ELENA  │          │
          │          │   M.   │          │
   ┌──────┴───────┐  │CRITICAL│   ┌──────┴───────┐
   │ ✝️ SPIRITUAL  │  │ ↓ TREND│   │ 📊 MONITOR  │
   └──────────────┘  └────────┘   └──────────────┘
```

- **Safety spoke** is solid and thicker (strokeWidth 3) to reinforce clinical priority.
- **All other spokes** use a dashed pattern.
- When a satellite is selected, non-selected satellites and their spokes fade to 35% opacity to draw the eye to the current selection.
- Clicking a satellite **expands** that category's recommendation cards (using `WorkflowNode` unchanged) in a panel below the hub.
- Clicking the same satellite again collapses the cards.
- Fully keyboard accessible: `role="button"`, `aria-pressed`, Enter/Space support.
- Below 480px container width a notice suggests switching to Classic view.

#### Coordinate system

All geometry is defined in a **500 × 500 SVG coordinate space**. The SVG element uses `viewBox="0 0 500 500"` with `preserveAspectRatio="xMidYMid meet"` so it scales proportionally. Satellite positions are expressed as CSS percentages derived from the same coordinate space via `toPct()`, so HTML elements and SVG lines remain perfectly aligned at any container width. The container uses `aspectRatio: 1` to keep the diagram circular.

#### Angle distribution

Satellites are distributed evenly around the hub at `360° / n` intervals where n is the number of active categories. Safety is anchored at `-π/2` (12-o'clock). Formula: `angle[i] = -π/2 + i × (2π / n)`.

### Priority Matrix (`WorkflowCanvasPriority.jsx`)

A **2D CSS Grid** triage view:

- **Rows (Y-axis):** priority tier — Critical (P9–10), High (P7–8), Medium (P5–6), Low (P1–4)
- **Columns (X-axis):** active recommendation category in `CATEGORY_ORDER`

```
             Safety   Clinical   Session   Monitoring   Spiritual
Critical   [ P10 ]
High                  [ P7 ]     [ P7 ]
Medium                             [ P5 ]
Low                                          [ P3 ]    [ P2 opt ]
```

Key properties:

- Only rows and columns that have at least one recommendation are rendered — no phantom empty rows.
- Empty individual cells display as subtle **dashed-border placeholders** to communicate "nothing here" without removing column context.
- Safety is always the leftmost column; Spiritual is always rightmost with muted styling.
- Each **mini-card** shows the recommendation title and a `P{n}` priority badge. A hover tooltip shows the full summary sentence.
- Clicking a mini-card opens the shared `RecommendationDrawer` for full detail and actions.
- The row-label column (`position: sticky; left: 0`) stays visible during horizontal scroll.
- A legend line above the grid explains the axes.

#### Grid rendering technique

All grid cells are built as a **flat array with explicit `gridRow` / `gridColumn` CSS properties** and rendered as direct children of a single CSS Grid container. This avoids the React Fragment-inside-grid problem where fragments do not map to grid tracks correctly, and keeps the layout logic readable and debuggable.

```
gridTemplateColumns: 140px repeat(n, minmax(160px, 1fr))
```

## Floating View-Cycle Button

A **floating `ActionIcon`** is anchored at `position: absolute; bottom: 20px; right: 20px` inside the canvas panel. It cycles through all available views on each click:

| State | Icon | Colour | Tooltip |
| --- | --- | --- | --- |
| Showing Classic List | `≡` | Gray | "Classic List — click for Radial Hub" |
| Showing Radial Hub | `⊙` | Blue | "Radial Hub — click for Priority Matrix" |
| Showing Priority Matrix | `⊞` | Violet | "Priority Matrix — click for Classic List" |

The selected variant is written to `localStorage` key `fw_view_variant` on every change and read back on mount, so the counselor's preference persists across page refreshes and sessions.

Adding a future fourth view requires only:
1. A new `WorkflowCanvas*.jsx` file.
2. Adding its key to the `VARIANTS` array in `FaithWorkflowsPage.jsx`.
3. Adding a conditional render block (same props as all other canvas variants).

## Safety Invariants (all views)

These are non-negotiable and enforced identically across all three canvas variants:

1. Safety recommendations always rendered first / most prominently.
2. Safety recommendations with `priority >= 9` cannot be hidden or deferred (enforced in `WorkflowNode`).
3. Spiritual recommendations always carry an "(Optional)" label.
4. No spiritual recommendation can outrank a safety or clinical recommendation.
5. AI-generated content carries the watermark: *"AI-assisted draft — final judgment belongs to the counselor."*
