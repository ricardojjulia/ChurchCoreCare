## Features shipped

### A1 — Multi-Tenant SaaS Phase 4

**Provisioning worker** (`apps/worker/src/provision.js`)
- Polls `tenant_provisioning` every 15 seconds using `FOR UPDATE SKIP LOCKED` (concurrent-safe)
- FSM: `queued → in_progress → completed | failed`
- Creates Stripe trial subscription on successful provision
- Graceful shutdown via `SIGTERM`

**Tenant query layer** (`apps/api/src/db/queries/tenants.js`)
- `getTenantBySlug`, `createTenantSlug`, `deactivateTenantSlug`
- `getTenantSubscription`, `upsertTenantSubscription`, `isSubscriptionActive`

**DB migrations** (`apps/api/src/db/migrate.js`)
- `tenant_slugs (slug PK, tenant_id, is_active, created_at)`
- `tenant_subscriptions (id, tenant_id UNIQUE, stripe_customer_id, stripe_subscription_id, stripe_price_id, status, trial_ends_at, current_period_end, canceled_at, created_at, updated_at)`

### A2 — Stripe Subscription Billing

**Stripe service** (`apps/api/src/lib/stripe.js`)
- Lazy Stripe client; plans: Solo $69/mo, Group $99/mo, Seat $39/mo
- Exports: `createCustomer`, `createSubscription`, `cancelSubscription`, `reactivateSubscription`, `getSubscription`, `createBillingPortalSession`, `createCheckoutSession`, `constructWebhookEvent`

**Webhook handler** (`apps/api/src/lib/billing-webhooks.js`)
- Handles: `customer.subscription.{created,updated,deleted}`, `invoice.paid`, `invoice.payment_failed`, `customer.subscription.trial_will_end`
- Maps Stripe status → internal: `trialing→trial`, `active→active`, `past_due→past_due`, `canceled→canceled`
- Upserts `tenant_subscriptions` on every event

**Routes in `apps/api/src/index.js`**
- `POST /webhooks/stripe` — raw body accumulation before CORS/auth, signature verification
- `GET /v1/billing/subscription` — returns plan status + trial expiry for tenant
- `POST /v1/billing/portal` — creates Stripe Billing Portal session, returns redirect URL

**Environment variables added to `.env.example`:**
```
STRIPE_SECRET_KEY=
STRIPE_WEBHOOK_SECRET=
STRIPE_PRICE_SOLO=
STRIPE_PRICE_GROUP=
STRIPE_PRICE_SEAT=
APP_BASE_URL=http://localhost:3000
```

### B1 — AI Session Note Drafting

**`apps/api/src/lib/ai-notes.js`**
- `buildPrompt(format, faithIntegrationLevel)` — exported for testing
- `draftSessionNote({ format, sessionContext, faithIntegrationLevel })` — calls `claude-sonnet-4-6`
- Formats: `SOAP`, `DAP`, `BIRP`, `FAITH_INTEGRATED`
- Faith prompting: scripture + spiritual practice blocks added for `preferred`/`actively_integrated`
- Never logs `sessionContext` (PHI)
- Returns `{ draft, format, model: 'claude-sonnet-4-6' }`

**Route** `POST /v1/clients/:id/progress-notes/draft`
- Gated by `AI_NOTES_ENABLED` env var (404 when disabled)
- Fetches `faith_integration_level` from `client_faith_profiles`
- Returns 503 on Anthropic errors (never exposes raw error)
- Emits audit event on every call

**`apps/web/src/components/ClinicalChart/AiNoteDraftPanel.jsx`**
- Format Select (SOAP / DAP / BIRP / Faith Integrated)
- Session context Textarea (maxLength 2000)
- Sparkles icon on "Generate Draft" button
- Collapse-based integration in `SessionNotesTab.jsx`

**DB migration:** `progress_notes.note_format VARCHAR(32) NULL`, `progress_notes.ai_draft_used BOOLEAN NOT NULL DEFAULT FALSE`

**Environment variables:**
```
ANTHROPIC_API_KEY=
AI_NOTES_ENABLED=false
```

**Tests:** `apps/api/test/ai-notes.test.mjs` — 8 tests covering API key guard, format validation, SOAP/faith prompt logic

### B3 — Insurance Eligibility Verification

**`apps/api/src/lib/eligibility.js`**
- `buildEligibilityRequest({ client, insurance, provider })` — constructs Stedi 270 request
- `parseEligibilityResponse(response)` — extracts `isEligible`, `planName`, `deductible`, `copay`, `coinsurance`, `outOfPocket`, `mentalHealthCovered`
- `verifyEligibility(...)` — sends 270, parses 271, stores encrypted result in `client_insurance`
- `getCachedEligibility(clientInsuranceId, tenantId)` — returns cached result

**Route** `POST /v1/clients/:id/verify-eligibility` — gated by `STEDI_API_KEY`

**DB migration on `client_insurance`:** `eligibility_response_enc TEXT NULL`, `eligibility_checked_at TIMESTAMP NULL`, `eligibility_status VARCHAR(32) NULL`
