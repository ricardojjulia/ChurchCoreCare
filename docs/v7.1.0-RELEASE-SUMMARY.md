# v7.1.0 Release Summary

**Released:** 2026-06-11
**Branch:** main (PRs #120 – #123)

---

## What Changed

### Ministry-language redesign

Every user-facing label that used generic clinical or software terminology was
renamed to language that fits a Christian counseling practice:

| Before | After |
|---|---|
| Clinical Chart | Care Notes |
| Faithful Workflows | Shepherd AI |
| Workspace Studio | Care Practice Setup |
| Therapy Groups | Care Groups |
| Staff | Staff & Access |
| Treatment Plan | Care Plan |
| Discharge Note | Completion Note |
| Clinical Cautions | Care Cautions |
| Clinical Relevance | Care Relevance |

All renames are i18n value changes only — no key names changed — so existing
translations remain valid and only the display strings differ.

### Collapsible grouped sidebar navigation

The sidebar is reorganized into collapsible sections instead of a flat list:

- **Admin:** Care Team · Clients · Practice · Ministry Tools · Portal
- **Counselor:** My Clients · My Schedule
- **Operations staff / Client:** unchanged flat root list

Section open/closed state persists across page reloads via
`localStorage('churchcore.sidebar.openSections')`. The section containing the
currently active view auto-expands on navigation.

### Platform admin multi-practice management

A `platform_admin` user lands on a practice-picker dashboard rather than the
standard care dashboard. Each practice card shows:

- Active and total client counts
- Active and total staff counts
- Plan type and timezone
- "Manage" button to enter the practice

Entering a practice injects an `x-target-tenant` header into all subsequent
API calls so the admin can browse any practice's data without a separate login.
An indigo context banner shows which practice is active and provides a one-click
exit back to the picker.

Backend: `GET /v1/platform/practices` (platform admin only, audited as
`platform.practices.read`). `callerTenant()` in `index.js` extended to honour
`x-target-tenant` only when `session.role === 'platform_admin'` and the value
matches `[a-z0-9_-]{1,64}`.

### i18n audit tooling

Two new scripts help identify and triage English text showing on non-English
locales:

- **`tests/e2e/i18n-audit.spec.mjs`** — Playwright page walker that sets the
  locale to `es-PR`, collects all visible text via a DOM TreeWalker, and
  classifies each node as missing-translation or likely-hardcoded. Writes a
  JSON report to `tests/e2e/reports/i18n-audit.json`. Never fails — discovery
  tool only.

- **`ops/i18n-add-missing-keys.mjs`** — reads the audit report, suggests
  dot-namespaced keys using namespace heuristics (auth, client, counselor,
  scheduling, …), patches `packages/i18n/src/index.js` with blank English
  entries, and adds blank entries to `apps/api/data/i18n/es-MX.json` for
  translator pickup. Supports `--dry-run`, `--only hardcoded`,
  `--only missing`.

## Bug Fixes

### Sidebar sections did not expand on click (PR #121)

Mantine v9 renamed the `Collapse` component's open-state prop from `in` to
`expanded`. The sidebar was passing `in={isOpen}` which was silently ignored.
All collapsible sections appeared permanently closed. Fixed by changing to
`expanded={isOpen}`.

### Time Tracking showed Unauthenticated for valid sessions (PR #123)

The session object from `resolveSession()` uses snake_case (`tenant_id`,
`staff_account_id`). The time-entries, licensure-goals, and supervisor-notes
handlers were referencing camelCase (`tenantId`, `userId`). Both were always
`undefined`, causing the auth guard to return `401 Unauthenticated` on every
request. 52 property references corrected.

### Shepherd AI label not showing in dashboard card (PR #122)

The `metrics.faithfulWorkflows` i18n key was missed in the ministry-language
rename pass. The top-right summary card on the Care Dashboard continued to
show "Faithful Workflows". Fixed in both the English source and the Spanish
seed.

## Upgrade Notes

No database migrations. No environment variable changes. No breaking API
changes. Deploy by merging to `main` — Vercel auto-deploys.

Clear browser localStorage key `churchcore.sidebar.openSections` if the
sidebar sections appear stuck in an unexpected state after the upgrade (stale
data from a previous session).
